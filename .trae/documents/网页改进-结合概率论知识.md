# 网页改进 — 结合概率论知识

## 目标

在网站各板块中，以**轻量、不突兀**的方式标注涉及的概率论知识点，让老师一眼看出"这里用了什么概率论知识"。

## 设计原则

1. **小而美** — 不改变现有布局，只在关键位置添加小标签/提示
2. **可折叠** — 默认收起只显示知识点名称，点击展开看简短说明
3. **统一风格** — 所有概率论标注使用统一的视觉样式（紫色标签 + 🎲 图标）

## 视觉规范

- 标签样式：`🎲 概率论：XXX`，紫色小药丸标签（`bg-purple-500/10 text-purple-400 border-purple-500/20`）
- 展开内容：1-2 句话，说明该概率论知识在本板块中的具体应用
- 位置：放在各板块 SectionWrapper 的 subtitle 下方，或关键组件的右上角

## 新增组件

### `ProbabilityTag.vue`（通用概率论标签组件）

```
位置：src/components/ProbabilityTag.vue
功能：可折叠的概率论知识标签
Props：
  - tag: string — 知识点名称（如"KL散度"、"正态分布"）
  - description: string — 展开后的简短说明
  - defaultExpanded?: boolean — 默认是否展开
```

渲染效果：
- 收起状态：`🎲 概率论：KL散度` （紫色小药丸）
- 展开状态：药丸 + 下方淡紫色卡片显示说明文字

## 各板块改进方案

### 1. 对齐剧场（AlignmentTheater）

**位置**：RLHFRoadmap 组件下方

| 知识点 | 说明 |
|--------|------|
| **条件概率** | P(偏好\|回答) — 奖励模型建模的是"给定回答，人类偏好的概率"，即条件概率分布 |
| **Bradley-Terry 模型** | 人类偏好排序建模：P(回答A优于B) = σ(r(A) - r(B))，将奖励差转化为概率 |

**实现**：在 RLHFRoadmap 下方添加两个 ProbabilityTag

---

### 2. 四模演武场（QuadModelArena）

**位置**：StepExplanation 组件下方

| 知识点 | 说明 |
|--------|------|
| **条件概率与采样** | Token生成 = 从条件概率分布 P(a_t\|s_t) 中采样，每个token是随机变量的一次实现 |
| **对数概率** | log P(a_t\|s_t) — 使用对数概率避免连乘下溢，且将乘法变加法便于优化 |

**实现**：在 StepExplanation 下方添加两个 ProbabilityTag

---

### 3. KL散度调节器（KLController）

**位置**：DistributionChart 上方（图表区域的标题旁）

| 知识点 | 说明 |
|--------|------|
| **KL散度** | D_KL(P\|\|Q) = ∫P(x)log(P(x)/Q(x))dx — 衡量两个概率分布的"距离"，是信息论与概率论的核心概念 |
| **正态分布** | Actor和Reference均建模为高斯分布 N(μ,σ²)，其PDF为 f(x) = (1/σ√2π)exp(-(x-μ)²/2σ²) |
| **期望** | KL散度本质是期望：E_P[log(P/Q)]，即在对数似然比下对分布P取期望 |

**实现**：在 DistributionChart 组件上方添加三个 ProbabilityTag

---

### 4. GAE回溯器（GAERetracer）

**位置**：ParamControls 组件下方（VarianceBiasIndicator 上方）

| 知识点 | 说明 |
|--------|------|
| **偏差-方差权衡** | GAE中λ控制偏差与方差的平衡：λ→0高偏差低方差（类似TD），λ→1低偏差高方差（类似MC），这是概率论中估计量的核心性质 |
| **马尔可夫性** | GAE依赖马尔可夫假设：未来状态只依赖当前状态，与历史无关，即 P(s_{t+1}\|s_t,a_t,s_{t-1},...) = P(s_{t+1}\|s_t,a_t) |
| **递推期望** | GAE的递推公式 A_t = δ_t + γλA_{t+1} 本质是条件期望的递推计算，利用动态规划求解 |

**实现**：在 ParamControls 下方、VarianceBiasIndicator 上方添加三个 ProbabilityTag

---

### 5. 训练模拟器（TrainingSimulator）

**位置**：ChartGrid 上方

| 知识点 | 说明 |
|--------|------|
| **重要性采样** | PPO的ratio = π_θ(a\|s)/π_old(a\|s) 是重要性采样权重，用旧策略下的样本来估计新策略的期望 |
| **随机梯度下降** | 训练过程中使用SGD/Adam优化器，本质是对期望损失函数的随机近似，每次用小批量样本估计梯度 |
| **大数定律** | 训练收敛的理论保证：当样本量足够大时，经验损失趋近于真实期望损失 |

**实现**：在 ChartGrid 上方添加三个 ProbabilityTag

---

### 6. Hero Section（首页）

**位置**：副标题下方、按钮上方

**不添加** — 首页保持简洁，概率论知识在各具体板块中展示即可。
