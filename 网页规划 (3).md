# PPO-Lab - AI驯化实验室 · 网页规划文档

> **项目定位**：一个沉浸式RLHF算法交互教学网站，让用户通过"驯化AI"的故事线，从零理解并亲手操控RLHF-PPO训练过程。
>
> **核心理念**：参考 Google Teachable Machine 和 TensorFlow Playground 的设计哲学——**让用户在"玩"中理解复杂的算法原理**。
>
> **目标用户**：概率论课程同学、对AI感兴趣的大学生、PPT汇报观众。

---

## 一、设计系统（Design System）

### 1.1 视觉风格

**主风格**：深色科技感 + HUD/FUI元素 + 教育友好

整体采用深色背景营造"实验室"氛围，同时保证文字可读性和教育场景的亲和力。不使用过于极端的赛博朋克风格，而是在深色基底上使用明亮、饱和的强调色来区分不同模型和概念。

**设计灵感来源**：
- TensorFlow Playground：简洁、功能驱动、实时反馈
- Google Teachable Machine：三段式布局、即时训练/推理切换、置信度可视化
- GAN Lab（Google）：实时数据流可视化、参数滑块、动画过渡

### 1.3 字体方案

```
标题字体：Inter / 思源黑体 (Source Han Sans)
  - 用途：页面标题、模块名称、导航
  - 特点：现代、清晰、科技感

正文字体：Inter / 思源黑体
  - 用途：正文、说明文字、按钮
  - 特点：与标题统一，保证一致性

代码/公式字体：JetBrains Mono / Fira Code
  - 用途：代码片段、参数值、数值显示
  - 特点：等宽、清晰区分字符

字号层级：
  - 页面大标题：36-48px, font-weight: 700
  - 模块标题：24-30px, font-weight: 600
  - 卡片标题：18-20px, font-weight: 600
  - 正文：14-16px, font-weight: 400
  - 辅助文字：12-13px, font-weight: 400
  - 数值/代码：14px, font-family: monospace
```

### 1.4 图标规范

- **严禁使用emoji作为图标**（如🧠🎮📊等）
- 使用SVG图标库：**Lucide Icons** 或 **Heroicons**
- 图标尺寸统一：24x24 viewBox
- 可点击元素必须添加 `cursor-pointer`
- 悬停效果：颜色变化 + 微弱发光（box-shadow），不使用scale变换避免布局偏移

### 1.5 动效规范

```
时长规范：
  - 微交互（按钮hover、颜色变化）：150-200ms
  - 面板展开/收起：200-300ms
  - 数据流动画：500-800ms
  - 页面切换/模块过渡：300-400ms
  - 加载动画：持续循环，提供暂停选项

缓动函数：
  - 进入动画：ease-out（快速出现，缓慢停止）
  - 退出动画：ease-in（缓慢开始，快速消失）
  - 循环动画：ease-in-out

性能要求：
  - 仅使用 transform 和 opacity 做动画（GPU加速）
  - 不动画 width/height/top/left 等触发重排的属性
  - 尊重 prefers-reduced-motion 媒体查询
  - 流光/粒子效果使用 Canvas 或 CSS，避免DOM操作
```

## 二、整体页面架构

### 2.1 导航设计

**固定顶部导航栏**，采用毛玻璃效果（backdrop-blur），悬浮于内容之上。

```
┌─────────────────────────────────────────────────────────────────┐
│  [PPO-Lab Logo]   [对齐剧场] [四模演武] [KL调节器] [GAE回溯]  │

│                                    进度指示器: ● ● ○ ○ ○        │
└─────────────────────────────────────────────────────────────────┘
```

**导航特性**：
- Logo在左，导航项居中或靠右
- 当前所在模块高亮显示（底部指示条 + 文字颜色变化）
- 右侧显示学习进度指示器（5个圆点，已完成/当前/未完成）
- 滚动时导航栏背景加深
- 移动端：汉堡菜单折叠

### 2.2 页面结构总览

整个网站采用**单页滚动（Single Page Scroll）**模式，用户从上到下依次经过各模块。每个模块占满一个或多个视口高度。

```
┌─────────────────────────────────────────────────────────────────┐
│  Section 0: Hero 英雄区（全屏）                                  │
│  ─────────────────────────────────────────────────────────────  │
│  Section 1: 对齐剧场（故事引入）                                 │
│  ─────────────────────────────────────────────────────────────  │
│  Section 2: 四模演武场（模型协作）                               │
│  ─────────────────────────────────────────────────────────────  │
│  Section 3: KL散度调节器（约束理解）                             │
│  ─────────────────────────────────────────────────────────────  │
│  Section 4: GAE回溯器（优势函数）                                │
│  ─────────────────────────────────────────────────────────────  │
│  Section 5: 训练模拟器（终极实操）                               │
│  ─────────────────────────────────────────────────────────────  │
│  Footer: 结语                                                   │
└─────────────────────────────────────────────────────────────────┘
```

**滚动行为**：
- 使用 `scroll-behavior: smooth`
- 每个Section进入视口时触发入场动画（fade-in + slide-up）
- 导航栏进度指示器随滚动位置更新
- 不使用 scroll-jacking（强制滚动），尊重用户控制权

```
│  │  β的作用机制（可展开查看详情）                            │   │
│  │                                                          │   │
│  │  从奖励函数看：                                           │   │
│  │  R_t = −β·(log P_actor − log P_ref)                     │   │
│  │  β越大 → 惩罚/奖励的"杠杆力"越强 → 分布被强拉向Ref        │   │
│  │                                                          │   │
│  │  从梯度更新看：                                           │   │
│  │  β增大 → KL项权重上升 → 参数更新更保守                    │   │
│  │  β减小 → KL约束变弱 → 参数更敢大步更新                    │   │
│  └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**交互细节**：
1. **滑块拖动**：拖动β滑块，Actor分布曲线实时平滑变化，KL散度数值实时更新
2. **快捷预设按钮**：三个按钮对应三种典型状态，点击后滑块动画滑动到对应位置
3. **分布曲线**：使用Canvas或SVG绘制，Reference分布固定不动（灰色半透明），Actor分布动态变化（蓝色填充）
4. **状态解读**：根据当前β值所在区间，自动显示对应的解读文字和状态图标
5. **公式联动**：页面侧边或下方显示公式 $R_t = -\beta \ln \frac{\pi_\theta(a_t|s_t)}{\pi_{ref}(a_t|s_t)}$，其中β值与滑块联动高亮
6. **β作用机制**：可展开的详情面板，展示β如何通过奖励函数和梯度更新影响Actor

## 七、交互设计核心原则总结

> 以下原则贯穿整个网站设计，是实现"可玩性高、互动性强、故事完整、可视化漂亮"的保障。

### 原则1：即时反馈（Immediate Feedback）
- 任何用户操作（滑块、按钮、点击）都应在200ms内产生视觉反馈
- 参考 TensorFlow Playground：拖动滑块时训练过程实时变化

### 原则2：渐进式披露（Progressive Disclosure）
- 信息分层次展示：先展示核心概念，点击/悬停展开细节
- 公式默认折叠，感兴趣的用户可展开查看完整推导

### 原则3：探索自由度（Freedom to Explore）
- 用户可以随时调整任何参数，不需要按固定顺序操作
- 提供"重置"功能，鼓励大胆实验
- 没有"错误"的操作，只有"不同"的结果

### 原则4：叙事驱动（Narrative-Driven）
- 每个模块都有明确的叙事目标
- 从"问题"到"理解"到"实操"，层层递进
- 终极训练模拟器是前面所有学习的综合应用

### 原则5：诚实可视化（Honest Visualization）
- 参考 Teachable Machine：展示模型的不确定性，而非隐藏它
- 优势函数有正有负，Loss有波动，KL散度可能失控——这些都是真实的
- 不美化失败，而是让用户理解为什么失败、如何调整
