### 1.2 各技术栈详细说明

#### 1.2.1 Vue 3 (Composition API + TypeScript)

**版本要求**：`^3.5`

**安装命令**：

```bash
npm install vue@^3.5
```

**配置方式**：

项目入口 `src/main.ts` 中创建应用实例：

```typescript
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import router from './router'
import App from './App.vue'
import './style.css'

const app = createApp(App)
app.use(createPinia())
app.use(router)
app.mount('#app')
```

Vite 配置 `vite.config.ts`：

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import tailwindcss from '@tailwindcss/vite'
import { resolve } from 'path'

export default defineConfig({
  plugins: [vue(), tailwindcss()],
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src'),
    },
  },
})
```

**本项目具体用途**：

- 使用 `<script setup lang="ts">` 语法编写所有组件
- 使用 Composition API（`ref`、`reactive`、`computed`、`watch`、`onMounted` 等）组织组件逻辑
- 使用 `defineProps`、`defineEmits`、`defineExpose` 定义组件接口
- 通过 `provide`/`inject` 实现跨层级组件通信（如 ECharts 实例共享）


**配置方式**：

通过 `useECharts` 组合式函数封装，统一管理实例生命周期：

```typescript
// src/composables/useECharts.ts
import { ref, onMounted, onUnmounted, type Ref } from 'vue'
import * as echarts from 'echarts/core'
import { LineChart, BarChart, HeatmapChart } from 'echarts/charts'
import { GridComponent, TooltipComponent, LegendComponent, VisualMapComponent } from 'echarts/components'
import { CanvasRenderer } from 'echarts/renderers'

echarts.use([LineChart, BarChart, HeatmapChart, GridComponent, TooltipComponent, LegendComponent, VisualMapComponent, CanvasRenderer])

export function useECharts(containerRef: Ref<HTMLElement | null>) {
  let chart: echarts.ECharts | null = null

  onMounted(() => {
    if (containerRef.value) {
      chart = echarts.init(containerRef.value, 'dark')
    }
  })

  onUnmounted(() => {
    chart?.dispose()
  })

  return { chart }
}
```

**本项目具体用途**：

- **KL散度调节器**：双正态分布曲线对比图（`DistributionChart.vue`）
- **GAE回溯器**：优势函数柱状图（`AdvantageChart.vue`）
- **训练模拟器**：7个实时图表

  - Critic Loss 曲线（Streaming Area Chart，紫色 `#A78BFA`）
  - KL散度曲线（Line Chart + 警戒线，琥珀色 `#F59E0B`）
  - Reward 得分曲线（Line Chart，绿色 `#34D399`）
  - Returns 曲线（Line Chart，青色 `#06B6D4`）


**配置方式**：

通过 `useAnimation` 组合式函数封装常用动画：

```typescript
// src/composables/useAnimation.ts
import { onUnmounted } from 'vue'
import gsap from 'gsap'

export function useAnimation() {
  const timelines: gsap.core.Timeline[] = []

  function createTimeline(options?: gsap.TimelineVars) {
    const tl = gsap.timeline(options)
    timelines.push(tl)
    return tl
  }

  onUnmounted(() => {
    timelines.forEach(tl => tl.kill())
  })

  return { gsap, createTimeline }
}
```

**本项目具体用途**：

- **四模演武场**：Token 飞入动画、数据流线动画、模型圆环脉冲效果

```typescript
// src/api/request.ts
import axios from 'axios'

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:8000',
  timeout: 10000,
})

request.interceptors.response.use(
  (response) => {
    const { success, data, error } = response.data
    if (!success) {
      return Promise.reject(error)
    }
    return data
  },

    return Promise.reject(error)
  }
)
```

- `POST /api/gae/step` — 单步 GAE 计算

#### 1.2.8 WebSocket（原生）

**版本要求**：浏览器原生 API，无需安装

**配置方式**：

封装为 `useWebSocket` 组合式函数：

```typescript
// src/composables/useWebSocket.ts
import { ref, onUnmounted } from 'vue'

export function useWebSocket(url: string) {
  const ws = ref<WebSocket | null>(null)
  const isConnected = ref(false)

  function connect() {
    ws.value = new WebSocket(url)
    ws.value.onopen = () => { isConnected.value = true }
    ws.value.onclose = () => { isConnected.value = false }
  }

  function send(data: object) {
    ws.value?.send(JSON.stringify(data))
  }

  onUnmounted(() => {
    ws.value?.close()
  })

  return { ws, isConnected, connect, send }
}
```

**本项目具体用途**：

- **训练模拟器**：通过 `ws://localhost:8000/ws/training` 建立连接
- 客户端发送动作：`start`、`pause`、`resume`、`step`、`reset`、`update_params`
- 服务端推送：`epoch`（训练指标）、`achievement`（成就解锁）、`status`（训练状态）、`error`（错误）
- 心跳机制：客户端每 30 秒发送 Ping，60 秒无响应则断开

## 第4章 路由设计

### 4.1 路由模式

项目采用 **HTML5 History 模式**（`createWebHistory`），配合单页滚动架构。

核心设计思路：

1. 所有板块在同一页面内垂直排列，用户滚动浏览
2. Vue Router 的主要作用是：
   - 支持直接链接到特定板块（如 `/alignment`）
   - 导航进度追踪（通过 `afterEach` 守卫更新 `appStore`）
   - 浏览器前进/后退支持
3. 点击导航项时，**平滑滚动**到对应 section，同时更新 URL

### 4.2 路由配置

```typescript
// src/router/index.ts
import { createRouter, createWebHistory, type RouteRecordRaw } from 'vue-router'

const routes: RouteRecordRaw[] = [
  {
    path: '/',
    name: 'home',
    component: () => import('@/App.vue'),
    children: [
      {
        path: '',
        name: 'hero',
        meta: { section: 'hero', title: 'PPO-Lab' },
      },
      {
        path: 'alignment',
        name: 'alignment',
        meta: { section: 'alignment', title: '对齐剧场' },
      },
      {
        path: 'quad-model',
        name: 'quad-model',
        meta: { section: 'quad-model', title: '四模演武场' },
      },
      {
        path: 'kl-controller',
        name: 'kl-controller',
        meta: { section: 'kl-controller', title: 'KL散度调节器' },
      },
      {
        path: 'gae-retracer',
        name: 'gae-retracer',
        meta: { section: 'gae-retracer', title: 'GAE回溯器' },
      },
      {
        path: 'training-simulator',
        name: 'training-simulator',
        meta: { section: 'training-simulator', title: '训练模拟器' },
      },
    ],
  },
]

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes,
  scrollBehavior(to, from, savedPosition) {
    if (to.hash) {
      return { el: to.hash, behavior: 'smooth' }
    }
    if (savedPosition) {
      return savedPosition
    }
    return { top: 0 }
  },
})

export default router
```

### 4.4 滚动行为


**行为说明**：

- **有 savedPosition 时**：恢复浏览器历史位置，支持前进/后退
- **默认**：滚动到顶部

### 4.5 导航守卫

#### afterEach 守卫 — 更新当前 section

```typescript
router.afterEach((to) => {
  const appStore = useAppStore()
  const section = to.meta.section as string | undefined
  if (section) {
    appStore.setCurrentSection(section)
  }
  document.title = to.meta.title
    ? `${to.meta.title} - PPO-Lab`
    : 'PPO-Lab'
})
```

**作用**：

1. 更新 `appStore.currentSection`，驱动导航栏高亮和进度指示器
2. 更新页面标题（`document.title`）

### 4.6 导航与滚动联动

#### 从导航栏点击 → 滚动 + 更新URL

```typescript
// AppHeader.vue 中的导航点击处理
function handleNavClick(sectionName: string) {
  const element = document.getElementById(sectionName)
  if (element) {
    element.scrollIntoView({ behavior: 'smooth' })
    router.push({ name: sectionName })
  }
}
```

#### 从滚动 → 更新URL + 导航高亮

```typescript
// 使用 useIntersection 组合式函数
// 各 SectionWrapper 进入视口时触发 enter 事件
// 父组件监听 enter 事件，更新路由和 appStore

function onSectionEnter(sectionName: string) {
  if (router.currentRoute.value.name !== sectionName) {
    router.replace({ name: sectionName })
  }
  appStore.setCurrentSection(sectionName)
}
```

**使用 `router.replace` 而非 `router.push`**：避免滚动时产生大量历史记录，确保前进/后退行为合理。

### 4.7 进度追踪

`appStore` 中的进度数据结构：

```typescript
// src/stores/app.ts
interface AppState {
  currentSection: string
  visitedSections: Set<string>
```

### 5.2 quadModelStore（四模演武场）

**职责**：管理四模演武场的Token生成状态机、步进数据、模型参数。

#### 5.2.1 状态机设计

```
        startGeneration()
 IDLE ──────────────────→ GENERATING
  ↑                         │    │
  │ reset()                 │    │ pauseGeneration()
  │                         │    ↓
  │                         │  PAUSED
  │                         │    │
  │                         │    │ resumeGeneration()
  │                         │    ↓
  │                    GENERATING
  │                         │
  │                         │ 所有Token生成完毕
  │                         ↓
  └──────────────────── COMPLETE
         reset()
```

**状态转换规则**：

| 当前状态 | Action | 目标状态 | 条件 |
|---------|--------|---------|------|
| `idle` | `startGeneration` | `generating` | prompt非空 |
| `generating` | `pauseGeneration` | `paused` | - |
| `generating` | `stepForward` (最后一步) | `complete` | `is_complete === true` |
| `paused` | `resumeGeneration` | `generating` | - |
| `paused` | `stepForward` (最后一步) | `complete` | `is_complete === true` |
| 任意 | `reset` | `idle` | - |

#### 5.2.2 State 定义

```typescript

import { quadModelApi } from '@/api/quadModel'
```

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `status` | `QuadModelStatus` | `'idle'` | 状态机当前状态 |
| `currentStep` | `number` | `0` | 当前已生成的Token步数 |
| `episode` | `GenerationEpisode \| null` | `null` | 完整的生成回合数据 |
| `steps` | `TokenStep[]` | `[]` | 已生成的Token步骤列表 |
| `prompt` | `string` | `'Hello'` | 输入提示词 |

```typescript

    return state.steps.length
  },

      return state.steps[state.currentStep - 1]
    }
    return null
  },

  isPlaying(state): boolean {
    return state.status === 'generating'
  },

  canStepForward(state): boolean {

```

| Getter | 返回类型 | 说明 |
|--------|---------|------|
| `totalSteps` | `number` | 已生成的总Token步数 |
| `currentStepData` | `TokenStep \| null` | 当前步骤的详细数据 |
| `isPlaying` | `boolean` | 是否正在自动生成 |
| `canStepForward` | `boolean` | 是否可以执行下一步 |

#### 5.2.4 Actions

```typescript
actions: {
  async startGeneration() {
    this.reset()
    this.status = 'generating'
    try {
      const res = await quadModelApi.generate({
        prompt: this.prompt,
        actor_params: this.actorParams,
        ref_params: this.refParams,
        kl_ctl: this.klCtl
      })
      if (res.success && res.data) {
        this.episode = res.data.episode
      }
    } catch (error) {
      this.status = 'idle'
      throw error
    }
  },

  pauseGeneration() {
    if (this.status === 'generating') {
      this.status = 'paused'
    }
  },

  resumeGeneration() {
    if (this.status === 'paused') {
      this.status = 'generating'
    }
  },

  async stepForward() {
    if (!this.canStepForward) return
    try {
      const currentTokens = this.steps.map(s => s.token)
      const res = await quadModelApi.step({
        prompt: this.prompt,
        current_tokens: currentTokens,
        actor_params: this.actorParams,
        ref_params: this.refParams,
        kl_ctl: this.klCtl
      })
      if (res.success && res.data) {
        this.steps.push(res.data.step)
        this.currentStep++
        if (res.data.is_complete) {
          this.status = 'complete'
        }
      }
    } catch (error) {
      throw error
    }
  },

  reset() {
    this.status = 'idle'
    this.currentStep = 0
    this.episode = null
    this.steps = []
  }
}
```

| Action | 参数 | 说明 |
|--------|------|------|
| `startGeneration` | 无 | 重置状态并请求完整Episode，切换到generating |
| `pauseGeneration` | 无 | 暂停自动生成 |
| `resumeGeneration` | 无 | 恢复自动生成 |
| `stepForward` | 无 | 请求下一步Token数据，若完成则切换到complete |
| `reset` | 无 | 重置所有状态到idle |

```typescript
actions: {
  async setBeta(newBeta: number) {

    await this.calculateKL()
  },

  async setDistribution(params: { ref?: { mean: number; std: number }; actor?: { mean: number; std: number } }) {
    if (params.ref) this.refDistribution = { ...params.ref }
    if (params.actor) this.actorDistribution = { ...params.actor }
    await this.calculateKL()
  },

  async calculateKL() {
    try {
      const res = await klApi.calculate({
        beta: this.beta,
        ref_distribution: this.refDistribution,
        actor_distribution: this.actorDistribution,
        x_range: [-4, 4],
        num_points: 200

        this.refCurve = res.data.ref_curve.curve
        this.actorCurve = res.data.actor_curve.curve
        this.interpretation = res.data.interpretation
        this.formula = res.data.formula
      }
    } catch (error) {
      throw error
    }
  },

  async applyPreset(preset: 'free' | 'standard' | 'strict') {
    const presets = {
      free: { beta: 0.01, actor: { mean: 0.8, std: 1.5 } },

    const p = presets[preset]
    this.beta = p.beta
    this.actorDistribution = p.actor
    await this.calculateKL()
  }
}
```

### 5.4 gaeStore（GAE回溯器）

**职责**：管理GAE回溯器的状态机、回溯动画、GAE计算数据。

#### 5.4.1 状态机设计

```
        startTrace()
 IDLE ──────────────────→ TRACING
  ↑                         │    │
  │ reset()                 │    │ pauseTrace()
  │                         │    ↓
  │                         │  PAUSED
  │                         │    │
  │                         │    │ resumeTrace() / stepBackward()

  │                         │  TRACING
  │                         │
  │                         │ 回溯到t=0
  │                         ↓
  └──────────────────── COMPLETE
```

**状态转换规则**：

| 当前状态 | Action | 目标状态 | 条件 |

#### 5.4.4 Actions

```typescript
actions: {
  async startTrace() {
    this.status = 'tracing'
    this.currentTraceIndex = this.tokens.length - 1
    try {
      const res = await gaeApi.compute({
        tokens: this.tokens,
        rewards: this.rewards,
        values: this.values,
        gamma: this.gamma,
        lambda: this.lambda
      })
      if (res.success && res.data) {
        this.result = res.data
      }
    } catch (error) {
      this.status = 'idle'
      throw error
    }
  },

  pauseTrace() {
    if (this.status === 'tracing') {
      this.status = 'paused'
    }
  },

  async stepBackward() {
    if (this.status !== 'paused' && this.status !== 'tracing') return
    if (this.currentTraceIndex < 0) return

    try {
      const nextAdvantage = this.currentTraceIndex < this.tokens.length - 1
        ? this.result!.advantages[this.currentTraceIndex + 1]
        : 0

      const res = await gaeApi.step({
        t: this.currentTraceIndex,
        tokens: this.tokens,
        rewards: this.rewards,
        values: this.values,
        gamma: this.gamma,
        lambda: this.lambda,
        next_advantage: nextAdvantage
      })
      if (res.success && res.data) {
        this.currentStepCalc = res.data.calculation
        this.currentTraceIndex--
        if (this.currentTraceIndex < 0) {
          this.status = 'complete'
        }
      }
    } catch (error) {
      throw error
    }
  },

  reset() {
    this.status = 'idle'
    this.result = null
    this.currentTraceIndex = -1
    this.currentStepCalc = null
  },

  setGamma(value: number) {
    this.gamma = Math.max(0.9, Math.min(1.0, value))
  },

  setLambda(value: number) {
    this.lambda = Math.max(0.0, Math.min(1.0, value))
  },

  setSpeed(speed: 1 | 2 | 4) {
    this.animationSpeed = speed
  },

  async computeGAE() {
    try {
      const res = await gaeApi.compute({
        tokens: this.tokens,
        rewards: this.rewards,
        values: this.values,
        gamma: this.gamma,
        lambda: this.lambda
      })
      if (res.success && res.data) {
        this.result = res.data
      }
    } catch (error) {
      throw error
    }
  }
}
```

### 7.1 四模演武场 Mock 数据

#### 7.1.1 计算参数

6 个 Token：`["I", "am", "an", "AI", "model", "assistant"]`

#### 7.1.2 逐步计算验证

**核心公式**：

- `kl_div = log_prob - ref_log_prob`
- `r_t = -kl_ctl × kl_div`（非最后 Token）
- `r_t = -kl_ctl × kl_div + R_global`（最后 Token）
- `A_t = r_t + γ × V_next - V_t`

**Step 0 — Token "I"**：

```
log_prob    = -1.2034
ref_log_prob= -1.1024
kl_div      = -1.2034 - (-1.1024) = -0.1010
V_t         = 0.4523
V_next      = 0.5187
r_t         = -0.1 × (-0.1010) = 0.0101
A_t         = 0.0101 + 0.99 × 0.5187 - 0.4523
           = 0.0101 + 0.5135 - 0.4523 = 0.0713
```

**Step 1 — Token "am"**：

```
log_prob    = -0.8921
ref_log_prob= -0.8543
kl_div      = -0.8921 - (-0.8543) = -0.0378
V_t         = 0.5187
V_next      = 0.4834
r_t         = -0.1 × (-0.0378) = 0.0038
A_t         = 0.0038 + 0.99 × 0.4834 - 0.5187
           = 0.0038 + 0.4786 - 0.5187 = -0.0363
```

**Step 2 — Token "an"**：

```
log_prob    = -1.5234
ref_log_prob= -1.4012
kl_div      = -1.5234 - (-1.4012) = -0.1222
V_t         = 0.4834
V_next      = 0.5512
```

### 7.2 KL 散度调节器 Mock 数据

#### 7.2.1 计算公式

两个正态分布之间的 KL 散度：

$$
KL(N_{ref} \| N_{actor}) = \log\frac{\sigma_{actor}}{\sigma_{ref}} + \frac{\sigma_{ref}^2 + (\mu_{ref} - \mu_{actor})^2}{2\sigma_{actor}^2} - \frac{1}{2}
$$

正态分布 PDF：

$$
f(x; \mu, \sigma) = \frac{1}{\sigma\sqrt{2\pi}} \exp\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)
$$

重叠面积通过数值积分计算：

$$
\text{overlap} = \int_{-\infty}^{+\infty} \min(f_{ref}(x),\ f_{actor}(x))\,dx
$$

```typescript
function generateNormalCurve(
  mean: number,
  std: number,
  xMin: number,
  xMax: number,
  numPoints: number
): { x: number; y: number }[] {
  const step = (xMax - xMin) / (numPoints - 1)
  const coefficient = 1 / (std * Math.sqrt(2 * Math.PI))
  const points: { x: number; y: number }[] = []

  for (let i = 0; i < numPoints; i++) {
    const x = xMin + i * step
    const exponent = -Math.pow(x - mean, 2) / (2 * std * std)
    const y = coefficient * Math.exp(exponent)
    points.push({ x: parseFloat(x.toFixed(4)), y: parseFloat(y.toFixed(4)) })
  }

  return points
}
```

```typescript
  const noise = () => (Math.random() - 0.5) * 0.06

    (2.0 * Math.exp(-0.15 * epoch) + noise()).toFixed(4)
  )
  const criticLoss = parseFloat(
    (1.5 * Math.exp(-0.12 * epoch) + noise()).toFixed(4)
  )
  const klDiv = parseFloat(
    (0.15 + 0.08 * Math.sin(0.5 * epoch) + noise() * 0.5).toFixed(4)
  )

    (0.3 + 0.5 * (1 - Math.exp(-0.2 * epoch)) + noise() * 0.3).toFixed(4)
  )

  const scale = 1.0 / (1 + 0.15 * (epoch - 1))
  const baseAdvantages = [2.6703, 2.1015, 2.0271, 1.5205, 1.1999, -0.1000]
  const baseValues = [0.4, 0.6, 0.5, 0.3, 0.8, 0.6]

  const advantages = baseAdvantages.map((a) =>
    parseFloat((a * scale + noise() * 0.1).toFixed(4))
  )
  const returns = advantages.map((a, i) =>
    parseFloat((a + baseValues[i] * scale).toFixed(4))
  )

  return {
    epoch,
    actor_loss: actorLoss,
    critic_loss: criticLoss,
    kl_div: klDiv,
    reward,
    advantages,
    returns
  }
}
```

### 7.5 Mock 数据生成逻辑说明

#### 7.5.1 各板块数据生成方法

| 板块 | 生成方法 | 关键算法 |
|------|---------|---------|
| 四模演武场 | 基于 Actor/Ref 高斯分布参数，逐 Token 计算 log_prob、kl_div、r_t、A_t | A1-A5 |
| KL 散度调节器 | 基于 Ref/Actor 正态分布参数，解析计算 KL 散度，数值积分计算重叠面积 | B1-B3 |
| GAE 回溯器 | 从后向前递推计算 δ_t 和 A_t，累加得到 advantages 和 returns | C1-C2 |
| 训练模拟器 | 指数衰减 + 正弦波动 + 高斯噪声模拟 PPO 训练曲线 | D1-D5 |

#### 7.5.2 四模演武场生成逻辑

```
输入：prompt, actor_params, ref_params, kl_ctl, R_global
对于每个 Token：
  1. log_prob ← Actor 分布的 log 概率（模拟值）
  2. ref_log_prob ← Ref 分布的 log 概率（模拟值）
  3. kl_div ← log_prob - ref_log_prob
  4. V_t ← Critic 预测（模拟值）
  5. V_next ← 下一状态 Critic 预测（模拟值）
  6. r_t ← -kl_ctl × kl_div（+ R_global 如果是最后 Token）
  7. A_t ← r_t + γ × V_next - V_t
输出：GenerationEpisode
```

#### 7.5.3 KL 散度调节器生成逻辑

```
输入：ref_params, actor_params, beta, x_range, num_points
1. 解析计算 KL(N_ref || N_actor)
2. 在 x_range 内均匀采样 num_points 个点
3. 对每个点计算 ref_pdf 和 actor_pdf
4. 数值积分计算 overlap_area = ∫ min(ref_pdf, actor_pdf) dx
5. 根据 KL 值和 beta 判断 interpretation 状态：
   - KL > 1.0 → warning（偏离过远）
   - 0.05 ≤ KL ≤ 1.0 → optimal（理想状态）
   - KL < 0.05 → locked（过度约束）
输出：KlCalculateResponse
```

#### 7.5.4 GAE 回溯器生成逻辑

```
输入：tokens, rewards, values, gamma, lambda
1. 初始化 A_{T} = 0（T 为序列长度）
2. 从 t = T-1 到 t = 0 反向遍历：
   a. V_next = (t < T-1) ? values[t+1] : 0
   b. δ_t = r_t + γ × V_next - V_t
   c. A_t = δ_t + γ × λ × A_{t+1}
3. advantages[i] = A_i（正向排列）
4. returns[i] = A_i + V_i
5. 根据 λ 值判断 varianceBias：
   - λ < 0.3：高偏差，低方差
   - 0.3 ≤ λ < 0.7：中等偏差，中等方差
   - λ ≥ 0.7：低偏差，高方差
输出：GAEResult
```

#### 7.5.5 训练模拟器生成逻辑

```
输入：epoch, hyperparams
1. actor_loss = A × exp(-α × epoch) + noise
   - A = 2.0（初始 Loss）
   - α = 0.15（衰减率，受 learning_rate 影响）
   - noise ~ N(0, 0.03)
2. critic_loss = B × exp(-β × epoch) + noise
   - B = 1.5, β = 0.12
3. kl_div = 0.15 + 0.08 × sin(0.5 × epoch) + noise
   - 均值受 kl_ctl 影响
   - 振幅受 clip_range 影响
4. reward = 0.3 + 0.5 × (1 - exp(-0.2 × epoch)) + noise
   - 渐近上限受 hyperparams 整体影响
5. advantages/returns 基于 GAE 基础值按 epoch 衰减缩放
输出：TrainingMetrics
```

#### 7.5.6 验证方法

| 验证项 | 方法 |
|--------|------|
| kl_div 一致性 | `kl_div === log_prob - ref_log_prob`（四模演武场） |
| r_t 一致性 | 非最后 Token：`r_t === -kl_ctl × kl_div`；最后 Token：`r_t === -kl_ctl × kl_div + R_global` |
| A_t 一致性 | `A_t === r_t + γ × V_next - V_t`（四模演武场） |
| GAE δ_t 一致性 | `δ_t === r_t + γ × V_{t+1} - V_t` |
| GAE A_t 递推 | `A_t === δ_t + γ × λ × A_{t+1}` |
| Returns 一致性 | `returns[i] === advantages[i] + values[i]` |
| KL 散度公式 | 手动代入 `log(σ₂/σ₁) + (σ₁² + (μ₁-μ₂)²)/(2σ₂²) - 0.5` 验证 |
| 训练曲线趋势 | actor_loss 单调递减、reward 单调递增（允许噪声波动） |

```typescript
apiClient.interceptors.response.use(
  (response: AxiosResponse<ApiResponse>) => {
    const { success, data, error } = response.data

    if (!success) {
      const apiError: ApiError = error || {
        code: 'UNKNOWN_ERROR',
        message: '未知错误'
      }
      console.error(`[API Error] ${apiError.code}: ${apiError.message}`)
      return Promise.reject(apiError)
    }

    return response
  },
  (error) => {
    if (error.response) {
      const status = error.response.status
      const errorMessages: Record<number, string> = {
        400: '请求参数错误',
        404: '请求资源不存在',
        422: '数据验证失败',
        500: '服务器内部错误'
      }
      const message = errorMessages[status] || `HTTP ${status} 错误`
      console.error(`[HTTP Error] ${status}: ${message}`)
      return Promise.reject({ code: `HTTP_${status}`, message })
    }

    if (error.code === 'ECONNABORTED') {
      console.error('[Timeout] 请求超时')
      return Promise.reject({ code: 'TIMEOUT', message: '请求超时，请检查网络连接' })
    }

    console.error('[Network Error]', error.message)
    return Promise.reject({ code: 'NETWORK_ERROR', message: '网络连接失败' })
  }
)
```

```typescript
  onAchievement: ((data: AchievementUnlock) => void) | null = null
  onStatus: ((data: WsStatusData) => void) | null = null
  onError: ((data: WsErrorData) => void) | null = null

  connect(url: string): void {
    this.url = url
    this.cleanup()

    const wsUrl = url.replace(/^http/, 'ws')
    this.ws = new WebSocket(wsUrl)

    this.ws.onopen = () => {
      console.log('[WS] 连接已建立')
      this.reconnectAttempts = 0
      this.startHeartbeat()
    }

    this.ws.onmessage = (event: MessageEvent) => {
      try {
        const message: WsServerMessage = JSON.parse(event.data)
        this.handleMessage(message)
      } catch (e) {
        console.error('[WS] 消息解析失败:', e)
      }
    }

    this.ws.onclose = (event: CloseEvent) => {
      console.log(`[WS] 连接关闭: code=${event.code}, reason=${event.reason}`)
      this.stopHeartbeat()

      if (!event.wasClean) {
        this.attemptReconnect()
      }
    }

    this.ws.onerror = () => {
      console.error('[WS] 连接错误')
    }
  }

  disconnect(): void {
    this.cleanup()
    this.reconnectAttempts = this.maxReconnectAttempts
  }

  send(message: WsClientMessage): void {
    if (this.ws && this.ws.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify(message))
    } else {
      console.warn('[WS] 连接未就绪，无法发送消息')
    }
  }

  startTraining(params: TrainingHyperparams): void {
    this.send({ action: 'start', params })
  }

  pauseTraining(): void {
    this.send({ action: 'pause' })
  }

  resumeTraining(): void {
    this.send({ action: 'resume' })
  }

  stepTraining(): void {
    this.send({ action: 'step' })
  }

  resetTraining(): void {
    this.send({ action: 'reset' })
  }

  updateParams(params: Partial<TrainingHyperparams>): void {
    this.send({ action: 'update_params', params })
  }

  private handleMessage(message: WsServerMessage): void {
    switch (message.type) {
      case 'epoch':
        if (this.onEpoch) {
          this.onEpoch(message.data as WsEpochData)
        }
        break
      case 'achievement':
        if (this.onAchievement) {
          this.onAchievement(message.data as WsAchievementData)
        }
        break
      case 'status':
        if (this.onStatus) {
          this.onStatus(message.data as WsStatusData)
        }
        break
      case 'error':
        if (this.onError) {
          this.onError(message.data as WsErrorData)
        }
        break
      case 'heartbeat_ack':
        break
    }
  }

  private startHeartbeat(): void {
    this.stopHeartbeat()
    this.heartbeatTimer = setInterval(() => {
      this.send({ action: 'heartbeat' })
    }, 30000)
  }

  private stopHeartbeat(): void {
    if (this.heartbeatTimer) {
      clearInterval(this.heartbeatTimer)
      this.heartbeatTimer = null
    }
  }

  private attemptReconnect(): void {
    if (this.reconnectAttempts >= this.maxReconnectAttempts) {
      console.error('[WS] 达到最大重连次数，停止重连')
      if (this.onError) {
        this.onError({
          code: 'WS_MAX_RECONNECT',
          message: 'WebSocket连接失败，请刷新页面重试'
        })
      }
      return
    }

    this.reconnectAttempts++
    const delay = this.reconnectDelay * this.reconnectAttempts

    console.log(`[WS] ${delay}ms 后尝试第 ${this.reconnectAttempts} 次重连...`)

    this.reconnectTimer = setTimeout(() => {
      this.connect(this.url)
    }, delay)
  }

  private cleanup(): void {
    this.stopHeartbeat()

    if (this.reconnectTimer) {
      clearTimeout(this.reconnectTimer)
      this.reconnectTimer = null
    }

    if (this.ws) {
      this.ws.onopen = null
      this.ws.onmessage = null
      this.ws.onclose = null
      this.ws.onerror = null

      if (
        this.ws.readyState === WebSocket.OPEN ||
        this.ws.readyState === WebSocket.CONNECTING
      ) {
        this.ws.close(1000, 'Client disconnect')
      }

      this.ws = null
    }
  }

  getReadyState(): number {
    return this.ws?.readyState ?? WebSocket.CLOSED
  }

  destroy(): void {
    this.disconnect()
    this.onEpoch = null
    this.onAchievement = null
    this.onStatus = null
    this.onError = null
  }
}
```

```typescript
  import.meta.env.VITE_WS_URL || 'ws://localhost:8000/ws/training'

function useWebSocket(): {
  wsInstance: Ref<TrainingWebSocket | null>
  isConnected: Ref<boolean>
  lastError: Ref<string | null>
  connect: () => void
  disconnect: () => void
} {
  const wsInstance: Ref<TrainingWebSocket | null> = ref(null)
  const isConnected: Ref<boolean> = ref(false)
  const lastError: Ref<string | null> = ref(null)

  let statusCheckTimer: ReturnType<typeof setInterval> | null = null

  function connect(): void {
    if (globalWs.instance) {
      wsInstance.value = globalWs.instance
      isConnected.value = globalWs.instance.getReadyState() === WebSocket.OPEN
      return
    }

    const ws = new TrainingWebSocket()

    ws.onStatus = (data: WsStatusData) => {
      isConnected.value = data.state === 'running' || data.state === 'paused'
    }

    ws.onError = (data: WsErrorData) => {
      lastError.value = `${data.code}: ${data.message}`
      isConnected.value = false
    }

    ws.connect(WS_URL)

    globalWs.instance = ws
    wsInstance.value = ws

    statusCheckTimer = setInterval(() => {
      if (wsInstance.value) {
        const state = wsInstance.value.getReadyState()
        isConnected.value = state === WebSocket.OPEN
      }
    }, 1000)
  }

  function disconnect(): void {
    if (globalWs.instance) {
      globalWs.instance.destroy()
      globalWs.instance = null
    }

    wsInstance.value = null
    isConnected.value = false
    lastError.value = null

    if (statusCheckTimer) {
      clearInterval(statusCheckTimer)
      statusCheckTimer = null
    }
  }

  onUnmounted(() => {
    if (statusCheckTimer) {
      clearInterval(statusCheckTimer)
    }
  })

  return {
    wsInstance,
    isConnected,
    lastError,
    connect,
    disconnect
  }
}
```

### 9.1 四模演武场：步进式动画的状态机设计

#### 9.1.1 状态机定义

四模演武场采用有限状态机（FSM）驱动Token生成的步进动画，状态流转如下：

```
IDLE → GENERATING → PAUSED → COMPLETE
  ↑        ↓           ↓        ↑
  └────────┘           └────────┘
```

**状态转换规则**：

| 当前状态 | 触发事件 | 目标状态 | 条件 |
|---------|---------|---------|------|
| `IDLE` | `startGeneration` | `GENERATING` | prompt非空 |
| `GENERATING` | `pause` | `PAUSED` | — |
| `PAUSED` | `resume` | `GENERATING` | — |
| `GENERATING` | `stepComplete` + `hasMoreSteps` | `GENERATING` | 继续下一步 |
| `GENERATING` | `stepComplete` + `noMoreSteps` | `COMPLETE` | 序列结束 |
| `COMPLETE` | `reset` | `IDLE` | — |
| `PAUSED` | `reset` | `IDLE` | — |

#### 9.1.2 状态机TypeScript实现

```typescript
// src/composables/useQuadModelStateMachine.ts
import { ref, computed, onUnmounted } from 'vue'
import { useQuadModelStore } from '@/stores/quadModel'
import { useAnimation } from '@/composables/useAnimation'
import { quadModelApi } from '@/api/quadModel'
import type { TokenStep } from '@/types'

type QuadModelState = 'idle' | 'generating' | 'paused' | 'complete'

interface SubStep {
  id: number
  label: string
  color: string
  duration: number
}

const SUB_STEPS: SubStep[] = [
  { id: 0, label: 'Actor生成Token', color: '#3B82F6', duration: 800 },
  { id: 1, label: 'Reference对比概率', color: '#F59E0B', duration: 600 },
  { id: 2, label: 'Critic预测价值', color: '#A78BFA', duration: 600 },
  { id: 3, label: '计算优势函数', color: '#34D399', duration: 700 },
]

export function useQuadModelStateMachine() {
  const store = useQuadModelStore()
  const { gsap, createTimeline } = useAnimation()

  const state = ref<QuadModelState>('idle')
  const currentTokenIndex = ref(0)
  const currentSubStep = ref(0)
  const isTransitioning = ref(false)

  const canStart = computed(() => state.value === 'idle')
  const canPause = computed(() => state.value === 'generating')
  const canResume = computed(() => state.value === 'paused')
  const canReset = computed(() => state.value !== 'idle')
  const canStepForward = computed(() =>
    state.value === 'generating' || state.value === 'paused'
  )
  const hasMoreSteps = computed(() => {
    if (!store.episode) return false
    return currentTokenIndex.value < store.episode.steps.length
  })
  const currentStepData = computed<TokenStep | null>(() => {
    if (!store.episode || currentTokenIndex.value <= 0) return null
    return store.episode.steps[currentTokenIndex.value - 1]
  })

  let autoPlayTimer: ReturnType<typeof setTimeout> | null = null
  let stepTimeline: gsap.core.Timeline | null = null

  function transition(newState: QuadModelState) {
    state.value = newState
    store.status = newState
  }

  async function startGeneration() {
    if (!canStart.value) return
    transition('generating')
    currentTokenIndex.value = 0
    currentSubStep.value = 0

    try {
      await store.startGeneration()
      if (store.episode && store.episode.steps.length > 0) {
        await executeNextStep()
      }
    } catch (error) {
      transition('idle')
      throw error
    }
  }

  function pause() {
    if (!canPause.value) return
    if (autoPlayTimer) {
      clearTimeout(autoPlayTimer)
      autoPlayTimer = null
    }
    if (stepTimeline) {
      stepTimeline.pause()
    }
    transition('paused')
  }

  function resume() {
    if (!canResume.value) return
    transition('generating')
    if (stepTimeline) {
      stepTimeline.resume()
    } else {
      executeNextStep()
    }
  }

  async function stepForward() {
    if (!canStepForward.value) return
    if (state.value === 'paused') {
      transition('generating')
    }
    await executeNextStep()
  }

  async function executeNextStep() {
    if (isTransitioning.value) return
    if (!hasMoreSteps.value) {
      transition('complete')
      return
    }

    isTransitioning.value = true
    currentTokenIndex.value++
    currentSubStep.value = 0

    stepTimeline = createStepTimeline(currentTokenIndex.value - 1)

    stepTimeline.eventCallback('onComplete', () => {
      isTransitioning.value = false
      if (state.value === 'generating' && hasMoreSteps.value) {
        autoPlayTimer = setTimeout(() => {
          executeNextStep()
        }, 300)
      } else if (!hasMoreSteps.value) {
        transition('complete')
      }
    })

    stepTimeline.play()
  }

  function createStepTimeline(stepIndex: number): gsap.core.Timeline {
    const tl = createTimeline({ paused: true })
    const stepData = store.episode!.steps[stepIndex]

    SUB_STEPS.forEach((sub, i) => {
      tl.call(() => { currentSubStep.value = i }, null, `+=0.1`)
      tl.to(`.model-node--${getModelName(i)}`, {
        scale: 1.15,
        boxShadow: `0 0 20px ${sub.color}80`,
        duration: 0.3,
        ease: 'power2.out',
      }, `+=0.05`)
      tl.to(`.model-node--${getModelName(i)}`, {
        scale: 1,
        boxShadow: 'none',
        duration: 0.2,
        ease: 'power2.in',
      }, `+=${sub.duration / 1000}`)

      if (i === 0) {
        tl.fromTo(`.token-chip--${stepIndex}`, {
          opacity: 0,
          y: -20,
          scale: 0.8,
        }, {
          opacity: 1,
          y: 0,
          scale: 1,
          duration: 0.5,
          ease: 'back.out(1.7)',
        }, '<0.2')
      }

      if (i === 1) {
        tl.fromTo(`.kl-value--${stepIndex}`, {
          opacity: 0,
          scale: 0.5,
        }, {
          opacity: 1,
          scale: 1,
          duration: 0.4,
          ease: 'back.out(2)',
        }, '<0.1')
      }

      if (i === 2) {
        tl.fromTo(`.v-value--${stepIndex}`, {
          opacity: 0,
          scale: 0.5,
        }, {
          opacity: 1,
          scale: 1,
          duration: 0.4,
          ease: 'back.out(2)',
        }, '<0.1')
      }

      if (i === 3) {
        tl.fromTo(`.a-value--${stepIndex}`, {
          opacity: 0,
          scale: 0.5,
        }, {
          opacity: 1,
          scale: 1,
          duration: 0.4,
          ease: 'back.out(2)',
        }, '<0.1')
      }
    })

    return tl
  }

  function getModelName(subStepIndex: number): string {
    const names = ['actor', 'reference', 'critic', 'reward']
    return names[subStepIndex]
  }

  function reset() {
    if (autoPlayTimer) {
      clearTimeout(autoPlayTimer)
      autoPlayTimer = null
    }
    if (stepTimeline) {
      stepTimeline.kill()
      stepTimeline = null
    }
    store.reset()
    transition('idle')
    currentTokenIndex.value = 0
    currentSubStep.value = 0
    isTransitioning.value = false
  }

  onUnmounted(() => {
    if (autoPlayTimer) clearTimeout(autoPlayTimer)
    if (stepTimeline) stepTimeline.kill()
  })

  return {
    state,
    currentTokenIndex,
    currentSubStep,
    isTransitioning,
    canStart,
    canPause,
    canResume,
    canReset,
    canStepForward,
    hasMoreSteps,
    currentStepData,
    subSteps: SUB_STEPS,
    startGeneration,
    pause,
    resume,
    stepForward,
    reset,
  }
}
```

#### 9.1.3 步进动画的GSAP时间线编排

```
1.6s ─── Critic预测价值 ────────────────────────

2.2s ─── 计算优势函数 ──────────────────────────

  2.4s   A_t数值弹出 + 公式展示

2.7s ─── 步骤完成 → 300ms间隔 → 下一步 ──────
```

#### 9.1.4 与quadModelStore交互

3. **API调用**：通过 `store.startGeneration()` 发起API请求，组合函数不直接调用API
4. **重置**：调用 `store.reset()` 清空Store数据，同时清空组合函数内部状态

### 9.3 GAE回溯器：从后向前回溯的动画控制逻辑

#### 9.3.1 状态机定义

```
IDLE → TRACING → PAUSED → COMPLETE
  ↑        ↓          ↓        ↑
  └────────┘          └────────┘
```

**状态转换规则**：

| 当前状态 | 触发事件 | 目标状态 | 条件 |
|---------|---------|---------|------|
| `IDLE` | `startTrace` | `TRACING` | 数据完整 |
| `TRACING` | `pause` | `PAUSED` | — |
| `PAUSED` | `resume` | `TRACING` | — |
| `TRACING` | 到达t=0 | `COMPLETE` | `currentTraceIndex === 0` |
| `COMPLETE` | `reset` | `IDLE` | — |
| `PAUSED` | `reset` | `IDLE` | — |

#### 9.3.2 useGaeTracer组合函数

```typescript
// src/composables/useGaeTracer.ts
import { ref, computed, watch, onUnmounted } from 'vue'
import { useGaeStore } from '@/stores/gae'
import { useAnimation } from '@/composables/useAnimation'
import type { GAEStep, StepCalculation } from '@/types'

type GaeTraceState = 'idle' | 'tracing' | 'paused' | 'complete'
type AnimationSpeed = 1 | 2 | 4

const SPEED_DURATION_MAP: Record<AnimationSpeed, number> = {
  1: 1500,
  2: 750,
  4: 375,
}

export function useGaeTracer() {
  const store = useGaeStore()
  const { gsap, createTimeline } = useAnimation()

  const traceState = ref<GaeTraceState>('idle')
  const speed = ref<AnimationSpeed>(1)
  const singleStepMode = ref(false)
  const displayedAdvantages = ref<number[]>([])
  const typewriterText = ref('')
  const isTyping = ref(false)

  const canStart = computed(() => traceState.value === 'idle' && store.tokens.length > 0)
  const canPause = computed(() => traceState.value === 'tracing')
  const canResume = computed(() => traceState.value === 'paused')
  const canReset = computed(() => traceState.value !== 'idle')
  const stepDuration = computed(() => SPEED_DURATION_MAP[speed.value])
  const currentTraceStep = computed<GAEStep | null>(() => {
    if (!store.result || store.currentTraceIndex < 0) return null
    return store.result.steps.find(s => s.t === store.currentTraceIndex) || null
  })

  let traceTimer: ReturnType<typeof setTimeout> | null = null
  let traceTimeline: gsap.core.Timeline | null = null
  let typewriterTimeline: gsap.core.Timeline | null = null

  function transition(newState: GaeTraceState) {
    traceState.value = newState
    store.status = newState
  }

  async function startTrace() {
    if (!canStart.value) return
    transition('tracing')
    displayedAdvantages.value = new Array(store.tokens.length).fill(0)
    store.currentTraceIndex = store.tokens.length - 1

    try {
      await store.startTrace()
      if (store.result) {
        await executeTraceStep()
      }
    } catch (error) {
      transition('idle')
      throw error
    }
  }

  function pause() {
    if (!canPause.value) return
    clearTimers()
    if (traceTimeline) traceTimeline.pause()
    if (typewriterTimeline) typewriterTimeline.pause()
    transition('paused')
  }

  function resume() {
    if (!canResume.value) return
    transition('tracing')
    if (traceTimeline) {
      traceTimeline.resume()
    } else {
      executeTraceStep()
    }
  }

  async function stepOnce() {
    if (traceState.value !== 'paused' && traceState.value !== 'idle') return
    if (traceState.value === 'idle') {
      await startTrace()
      singleStepMode.value = true
      return
    }
    singleStepMode.value = true
    transition('tracing')
    await executeTraceStep()
  }

  async function executeTraceStep() {
    if (store.currentTraceIndex < 0) {
      transition('complete')
      return
    }

    const t = store.currentTraceIndex
    const stepData = store.result!.steps.find(s => s.t === t)
    if (!stepData) {
      transition('complete')
      return
    }

    try {
      await store.stepBackward()
    } catch (error) {
      transition('idle')
      throw error
    }

    traceTimeline = createBarGrowTimeline(t, stepData)
    typewriterTimeline = createTypewriterTimeline(stepData)

    traceTimeline.eventCallback('onComplete', () => {
      if (singleStepMode.value) {
        singleStepMode.value = false
        transition('paused')
        return
      }

      if (store.currentTraceIndex < 0) {
        transition('complete')
      } else {
        traceTimer = setTimeout(() => {
          executeTraceStep()
        }, 200)
      }
    })

    traceTimeline.play()
    typewriterTimeline.play()
  }

  function createBarGrowTimeline(
    tokenIndex: number,
    stepData: GAEStep
  ): gsap.core.Timeline {
    const tl = createTimeline({ paused: true })
    const targetValue = stepData.A_t
    const barColor = stepData.is_positive ? '#22C55E' : '#EF4444'

    tl.fromTo(`.advantage-bar--${tokenIndex}`, {
      scaleY: 0,
      backgroundColor: barColor,
    }, {
      scaleY: 1,
      duration: stepDuration.value / 1000 * 0.6,
      ease: 'power2.out',
      onUpdate: () => {
        const progress = tl.progress()
        displayedAdvantages.value[tokenIndex] = targetValue * progress
      },
    }, 0)

    tl.to(`.advantage-bar--${tokenIndex}`, {
      filter: 'brightness(1.3)',
      duration: 0.15,
      ease: 'power2.in',
    }, 0)

    tl.to(`.advantage-bar--${tokenIndex}`, {
      filter: 'brightness(1)',
      duration: 0.3,
      ease: 'power2.out',
    }, stepDuration.value / 1000 * 0.6)

    tl.to(`.trace-indicator`, {
      left: `${(tokenIndex / (store.tokens.length - 1)) * 100}%`,
      duration: stepDuration.value / 1000 * 0.3,
      ease: 'power2.inOut',
    }, 0)

    return tl
  }

  function createTypewriterTimeline(stepData: GAEStep): gsap.core.Timeline {
    const tl = createTimeline({ paused: true })
    const calc = store.currentStepCalc
    if (!calc) return tl

    const fullText = buildCalculationText(stepData, calc)
    const textProxy = { charIndex: 0 }

    tl.to(textProxy, {
      charIndex: fullText.length,
      duration: stepDuration.value / 1000 * 0.8,
      ease: 'none',
      onUpdate: () => {
        typewriterText.value = fullText.substring(0, Math.floor(textProxy.charIndex))
      },
    }, 0)

    return tl
  }

  function buildCalculationText(stepData: GAEStep, calc: StepCalculation): string {
    const lines: string[] = []
    lines.push(`t=${stepData.t} Token="${stepData.token}"`)
    lines.push(`δ_${stepData.t} = ${calc.formulas.td_error.substitution}`)
    lines.push(`    = ${calc.formulas.td_error.result.toFixed(4)}`)
    lines.push(`A_${stepData.t} = ${calc.formulas.gae.substitution}`)
    lines.push(`    = ${calc.formulas.gae.result.toFixed(4)}`)
    return lines.join('\n')
  }

  function setSpeed(newSpeed: AnimationSpeed) {
    speed.value = newSpeed
    store.setSpeed(newSpeed)
  }

  function reset() {
    clearTimers()
    if (traceTimeline) { traceTimeline.kill(); traceTimeline = null }
    if (typewriterTimeline) { typewriterTimeline.kill(); typewriterTimeline = null }
    store.reset()
    transition('idle')
    displayedAdvantages.value = []
    typewriterText.value = ''
    isTyping.value = false
    singleStepMode.value = false
  }

  function clearTimers() {
    if (traceTimer) {
      clearTimeout(traceTimer)
      traceTimer = null
    }
  }

  watch([() => store.gamma, () => store.lambda], async () => {
    if (traceState.value !== 'idle') {
      reset()
    }
    await store.computeGAE()
  })

  onUnmounted(() => {
    clearTimers()
    if (traceTimeline) traceTimeline.kill()
    if (typewriterTimeline) typewriterTimeline.kill()
  })

  return {
    traceState,
    speed,
    singleStepMode,
    displayedAdvantages,
    typewriterText,
    isTyping,
    canStart,
    canPause,
    canResume,
    canReset,
    stepDuration,
    currentTraceStep,
    startTrace,
    pause,
    resume,
    stepOnce,
    setSpeed,
    reset,
  }
}
```

```typescript
function handleEpochData(data: TrainingMetrics) {
  store.history.push(data)

  // 触发ECharts图表增量更新
  // 触发成就判定
}

function handleAchievement(data: AchievementUnlock) {
  const exists = store.achievements.some(a => a.achievement === data.achievement)
  if (!exists) {
    store.achievements.push(data)
    // 触发GSAP成就解锁动画
  }
}

function handleStatus(data: { state: TrainingStatus; epoch: number; max_epochs: number }) {
  store.trainingStatus = data.state
  store.currentEpoch = data.epoch
  store.maxEpochs = data.max_epochs
```

**WebSocket连接生命周期**：

```
1. 用户点击"开始训练" → connect() → 建立WebSocket连接
2. 连接成功 → startTraining() → 发送 {action: 'start', params: hyperparams}
3. 训练中 → 接收 epoch 消息 → 更新图表
4. 训练中 → 接收 achievement 消息 → 播放成就动画
5. 用户暂停 → pauseTraining() → 发送 {action: 'pause'}
6. 用户恢复 → resumeTraining() → 发送 {action: 'resume'}
7. 训练完成 → 接收 status: {state: 'completed'}
8. 用户重置 → resetTraining() → 发送 {action: 'reset'}
9. 页面卸载 → disconnect() → 关闭连接
```

```typescript
let heartbeatTimer: ReturnType<typeof setInterval> | null = null
let lastPongTime = 0
const HEARTBEAT_INTERVAL = 30000
const HEARTBEAT_TIMEOUT = 60000

function startHeartbeat() {
  heartbeatTimer = setInterval(() => {
    if (ws.value && isConnected.value) {
      ws.value.send(JSON.stringify({ type: 'ping' }))
      if (Date.now() - lastPongTime > HEARTBEAT_TIMEOUT) {
        reconnect()
      }
    }
  }, HEARTBEAT_INTERVAL)
}

function stopHeartbeat() {
  if (heartbeatTimer) {
    clearInterval(heartbeatTimer)
    heartbeatTimer = null
  }
}
```

#### 9.4.4 成就判定和动画

**5个成就的判定逻辑**：

| 成就 | 类型 | 判定条件 | 判定时机 |
|------|------|---------|---------|
| 首次收敛 | `first_converge` | `actor_loss < 0.5`（首次出现） | 每个epoch消息后 |
| KL稳定 | `kl_stable` | 最近10个epoch的 `kl_div` 均在 [0.1, 0.3] | 每个epoch消息后 |
| 连续优化 | `continuous_improve` | 最近20个epoch的 `actor_loss` 严格递减 | 每个epoch消息后 |
| 精准调参 | `precise_tuning` | 最新epoch `actor_loss < 0.3` 且 `kl_div < 0.2` | 每个epoch消息后 |
| 驯化大师 | `master` | 最新epoch `actor_loss < 0.1` 且 `kl_div < 0.2` 且 `reward > 0.8` | 每个epoch消息后 |

#### 10.4.1 容器resize自动调整

使用 `ResizeObserver` 监听容器尺寸变化，自动调用 `chart.resize()`：

```typescript
// src/composables/useECharts.ts — 增强版
import { ref, onMounted, onUnmounted, type Ref } from 'vue'
import * as echarts from 'echarts/core'

export function useECharts(containerRef: Ref<HTMLElement | null>) {
  let chart: echarts.ECharts | null = null
  let resizeObserver: ResizeObserver | null = null

  onMounted(() => {
    if (containerRef.value) {
      chart = echarts.init(containerRef.value, 'ppo-lab')

      resizeObserver = new ResizeObserver((entries) => {
        for (const entry of entries) {
          if (entry.contentRect.width > 0 && entry.contentRect.height > 0) {
            chart?.resize()
          }
        }
      })
      resizeObserver.observe(containerRef.value)
    }
  })

  onUnmounted(() => {
    resizeObserver?.disconnect()
    chart?.dispose()
  })

  function setOption(option: echarts.EChartsOption, notMerge: boolean = false) {
    chart?.setOption(option, notMerge)
  }

  return { chart, setOption }
}
```

```typescript
// src/utils/chart.ts — 降采样函数
function downsampleData<T extends { epoch: number; value: number }>(
  data: T[],
  maxPoints: number = 100
): T[] {
  if (data.length <= maxPoints) return data

  const step = data.length / maxPoints
  const result: T[] = []

  for (let i = 0; i < maxPoints; i++) {
    const idx = Math.floor(i * step)
    result.push(data[idx])
  }

  const lastPoint = data[data.length - 1]
  if (result[result.length - 1] !== lastPoint) {
    result[result.length - 1] = lastPoint
  }

  return result
}
```

**降采样策略**：

| 数据量 | 策略 | 说明 |
|--------|------|------|
| ≤ 100 | 不降采样 | 全量显示 |
| 101-500 | 均匀采样至100点 | 保留首尾点 |
| > 500 | 均匀采样至100点 + 关闭动画 | 性能优先 |

```typescript
// src/composables/useLazyChart.ts
import { ref, onMounted, onUnmounted, type Ref } from 'vue'
import { useIntersection } from '@/composables/useIntersection'

export function useLazyChart(containerRef: Ref<HTMLElement | null>) {
  const isVisible = ref(false)
  const isInitialized = ref(false)
  let chart: echarts.ECharts | null = null

  const { observe, unobserve } = useIntersection({
    threshold: 0.1,
    onEnter: () => {
      isVisible.value = true
      if (!isInitialized.value && containerRef.value) {
        chart = echarts.init(containerRef.value, 'ppo-lab')
        isInitialized.value = true
      }
    },
    onLeave: () => {
      isVisible.value = false
    },
  })

  onMounted(() => {
    if (containerRef.value) {
      observe(containerRef.value)
    }
  })

  onUnmounted(() => {
    if (containerRef.value) {
      unobserve(containerRef.value)
    }
    chart?.dispose()
  })

  return { chart, isVisible, isInitialized }
}
```

**懒加载策略**：图表容器进入视口10%时才初始化ECharts实例，避免首屏加载9个图表实例的性能开销。

```typescript
// 图表增量更新策略：使用 setOption 的 notMerge=false（默认值）
// 新数据点追加到已有数据，不重建整个图表

function appendDataPoint(
  chart: echarts.ECharts,
  epoch: number,
  value: number,
  maxPoints: number = 100
) {
  const option = chart.getOption() as any
  const existingData = option.series[0].data || []
  const existingXAxis = option.xAxis[0].data || []

  existingData.push(value)
  existingXAxis.push(epoch)

  if (existingData.length > maxPoints) {
    existingData.shift()
    existingXAxis.shift()
  }

  chart.setOption({
    xAxis: { data: existingXAxis },
    series: [{ data: existingData }],
  })
}
```

**增量更新 vs 全量更新**：

| 方式 | API | 性能 | 适用场景 |
|------|-----|------|---------|
| 增量更新 | `setOption({series:[{data}]}), notMerge=false` | 高 | Streaming图表（Loss/KL/Reward/Returns） |
| 全量替换 | `setOption(fullOption, notMerge=true)` | 低 | 优势函数分布、热力图（数据结构变化） |
