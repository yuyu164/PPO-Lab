## 一、核心算法总览

### 1.1 必须实现的算法清单

| 板块 | 必须实现的算法 | 禁止行为 |
|------|---------------|----------|
| **四模演武场** | Token生成流程、KL散度计算、Reward计算、价值函数 | 禁止随机生成假数据 |
| **KL散度调节器** | KL散度公式、PPO-Penalty约束机制 | 禁止预设固定曲线 |
| **GAE回溯器** | GAE算法（从后向前DP）、TD error计算 | 禁止随机生成优势值 |
| **训练模拟器** | PPO-Clip Actor Loss、Critic Loss、PPO训练循环 | 禁止预设Loss曲线 |

### 1.2 核心公式速查

```
【策略梯度】
∇J(π_θ) = E_{τ∼π_θ}[R(τ)∑_t ∇log π_θ(a_t|s_t)]

【状态价值函数】
V_π(s_t) = E_π[G_t|s_t] = E_π[r_t + γV_π(s_{t+1})|s_t]

【动作价值函数】
Q_π(s_t, a_t) = E_π[G_t|s_t, a_t] = E_{s_{t+1}}[r_t + γV_π(s_{t+1})]

【优势函数】
A_π(s_t, a_t) = Q_π(s_t, a_t) - V_π(s_t)

【TD error】
δ_t = r_t + γV_π(s_{t+1}) - V_π(s_t)

【V和Q的关系】
V_π(s_t) = ∑_{a_t} π(a_t|s_t) Q_π(s_t, a_t)

【GAE公式】
Ψ_t = ∑_{l=0}^∞ (γλ)^l δ_{t+l}
其中 δ_t = r_t + γV_π(s_{t+1}) - V_π(s_t)

【GAE递归形式】
A_t^{GAE} = δ_t + γλ·A_{t+1}^{GAE}

【Returns计算】
R_t = A_t^{GAE} + V_t^{old}

【重要性采样后的Actor优化目标】
J(π_θ) = E_t[π_θ(a_t|s_t)/π_old(a_t|s_t) · A_φ^{GAE}(s_t, a_t)]

【PPO-Clip Actor Loss】
J^{CLIP}(π_θ) = E_t[min[r_t(θ)·A, clip(r_t(θ), 1-ε, 1+ε)·A]]
其中 r_t(θ) = π_θ(a_t|s_t)/π_old(a_t|s_t)

【PPO-Penalty】
J(π_θ) = E_t[ratio·A - β·KL(π_old, π_θ)]

【Critic Loss】
L(V_φ) = E_t[max[(V_t^{new} - R_t)², (V_t^{CLIP} - R_t)²]]
其中 V_t^{CLIP} = clip(V_t^{new}, V_t^{old} - ε, V_t^{old} + ε)

【R_t设计（DeepSpeed-Chat实现）】
当 t ≠ T: R_t = -kl_ctl * log[π_θ(a_t|s_t)/π_ref(a_t|s_t)]
当 t = T: R_t = -kl_ctl * log[π_θ(a_t|s_t)/π_ref(a_t|s_t)] + R_global
```

## 二、四模演武场 - 算法规范

### 2.1 功能描述
展示Actor、Critic、Reward、Reference四个模型的协作流程，通过步进式动画演示Token生成过程。

### 2.2 必须实现的算法

#### 算法A1: Token生成流程

**输入：**
- `prompt`: 输入提示词（字符串，如"Hello"）
- `actor_params`: Actor模型参数（简化版，可用正态分布模拟）
- `ref_params`: Reference模型参数（固定，不更新）

**输出：**
- `tokens`: 生成的Token序列（数组，如["I", "am", "an", "AI"])
- `log_probs`: Actor对每个Token的log概率（数组）
- `ref_log_probs`: Reference对每个Token的log概率（数组）

**计算逻辑：**
```javascript
// 简化模拟：用正态分布采样生成Token概率
function generateToken(context, actorParams) {
    // 基于当前上下文，计算词表中每个Token的概率
    // 简化：使用softmax over随机初始化的logits
    const logits = randomNormal(vocabSize, actorParams.mean, actorParams.std);
    const probs = softmax(logits);
    const token = sampleFromDistribution(probs);
    const logProb = Math.log(probs[token]);
    return { token, logProb };
}
```

#### 算法A2: KL散度计算

**输入：**
- `log_probs`: Actor的log概率数组
- `ref_log_probs`: Reference的log概率数组

**输出：**
- `kl_divs`: 每个Token的KL散度值（数组）

**计算逻辑：**
```javascript
function calculateKL(log_probs, ref_log_probs) {
    // KL散度近似：log_probs - ref_log_probs
    // 来源：知乎文章8.6节 PPO-Penalty
    return log_probs.map((lp, i) => lp - ref_log_probs[i]);
}
```

#### 算法A3: Reward计算（R_t设计）

**输入：**
- `kl_divs`: KL散度数组
- `kl_ctl`: KL惩罚系数（用户可调，默认0.1）
- `R_global`: 全局奖励（由模拟的Reward Model给出）
- `isLastToken`: 是否为序列最后一个Token

**输出：**
- `rewards`: 每个Token的即时奖励（数组）

**计算逻辑（来源：报告4.3节）：**
```javascript
function calculateRewards(kl_divs, kl_ctl, R_global, isLastToken) {
    return kl_divs.map((kl, i) => {
        const klPenalty = -kl_ctl * kl;
        if (isLastToken[i]) {
            return klPenalty + R_global;  // 最后时刻加入全局奖励
        }
        return klPenalty;  // 中间时刻只有KL惩罚
    });
}
```

#### 算法A4: Critic价值预测（状态价值函数）

**来源：知乎文章第6节**

**输入：**
- `context`: 当前上下文（状态s_t）
- `criticParams`: Critic模型参数

**输出：**
- `V_t`: 预测的状态价值（标量）

**计算逻辑：**
```javascript
function predictValue(context, criticParams) {
    // 状态价值函数：V_π(s_t) = E_π[r_t + γV_π(s_{t+1})|s_t]
    // 简化：Critic输出一个标量价值
    const features = extractFeatures(context);
    return dotProduct(features, criticParams.weights) + criticParams.bias;
}
```

#### 算法A5: 优势函数计算

**来源：知乎文章第6.6节**

**输入：**
- `r_t`: 即时奖励
- `V_t`: 当前状态价值
- `V_next`: 下一状态价值

**输出：**
- `A_t`: 优势值

**计算逻辑：**
```javascript
function calculateAdvantage(r_t, V_t, V_next, gamma) {
    // TD error: δ_t = r_t + γV(s_{t+1}) - V(s_t)
    // 优势函数：A = Q - V = E[δ_t]
    const td_error = r_t + gamma * V_next - V_t;
    return td_error;
}
```

## 三、KL散度调节器 - 算法规范

### 3.1 功能描述
用户调节KL惩罚系数β，理解PPO-Penalty中的KL约束机制。

### 3.2 必须实现的算法

#### 算法B1: PPO-Penalty约束机制

**来源：知乎文章第8.6节**

**核心概念：**
PPO-Penalty将KL散度约束直接放入优化目标中：
```
J(π_θ) = E_t[ratio·A - β·KL(π_old, π_θ)]
```

**β的动态调整策略：**
- 当 KL ≥ KL_max 时，增大β，把分布拉回来
- 当 KL ≤ KL_min 时，减小β，降低KL散度一项的影响

**输入：**
- `beta`: KL惩罚系数（范围0.01-1.0）

**输出：**
- `interpretation`: 状态解读

**计算逻辑：**
```javascript
function interpretBeta(beta) {
    // 来源：知乎文章8.6节
    // β控制KL散度在奖励中的权重
    if (beta < 0.05) {
        return {
            status: "warning",
            title: "β值过低",
            description: "KL惩罚过小，Actor可能偏离Reference过远，导致Reward Hacking"
        };
    } else if (beta >= 0.05 && beta <= 0.2) {
        return {
            status: "optimal",
            title: "理想状态",
            description: "Actor在探索与约束之间取得平衡"
        };
    } else {
        return {
            status: "locked",
            title: "约束过强",
            description: "KL惩罚过大，Actor被过度压制，难以学习"
        };
    }
}
```

#### 算法B2: KL散度数值计算（两个正态分布之间）

**输入：**
- `mu1`, `sigma1`: 分布1参数（Reference）
- `mu2`, `sigma2`: 分布2参数（Actor）

**输出：**
- `kl_divergence`: KL散度值

**计算逻辑（标准数学公式）：**
```javascript
function calculateKLGaussian(mu1, sigma1, mu2, sigma2) {
    // 两个正态分布之间的KL散度解析公式
    // KL(N1||N2) = log(sigma2/sigma1) + (sigma1² + (mu1-mu2)²)/(2·sigma2²) - 0.5
    const ratio = sigma2 / sigma1;
    const diff = mu1 - mu2;
    const kl = Math.log(ratio) + (sigma1**2 + diff**2) / (2 * sigma2**2) - 0.5;
    return kl;
}
```

#### 算法B3: 分布曲线生成

**输入：**
- `mean`, `std`: 分布参数
- `xRange`: x轴范围（如[-4, 4]）
- `numPoints`: 采样点数

**输出：**
- `curvePoints`: 曲线点数组 [{x, y}]

**计算逻辑：**
```javascript
function generateNormalCurve(mean, std, xRange, numPoints) {
    const points = [];
    const step = (xRange[1] - xRange[0]) / numPoints;
    
    for (let i = 0; i <= numPoints; i++) {
        const x = xRange[0] + i * step;
        const y = (1 / (std * Math.sqrt(2 * Math.PI))) * 
                  Math.exp(-0.5 * ((x - mean) / std) ** 2);
        points.push({ x, y });
    }
    
    return points;
}
```

### 3.3 重要说明：β与分布变化的关系

> **注意：知乎文章中没有给出"β如何具体影响Actor分布参数"的解析公式。**
>
> 文章只说明了：
> - β控制KL散度在奖励中的权重
> - β越大，KL惩罚越重，Actor越接近Reference
> - β越小，KL惩罚越轻，Actor可能偏离Reference
>
> **可视化实现方案：**
> - 方案1：使用正态分布KL散度的解析解，通过调整分布参数来展示β的影响
> - 方案2：直接展示PPO-Penalty的优化目标公式，让用户理解β的作用机制
>
> **推荐方案2**，因为更符合理论原理。

## 四、GAE回溯器 - 算法规范

### 4.1 功能描述
展示广义优势估计（GAE）的从后向前动态规划计算过程。

### 4.2 理论来源

**来源：知乎文章第8.3节**

GAE通过超参数λ控制方差-偏差的平衡：

```
Ψ_t = ∑_{l=0}^∞ (γλ)^l δ_{t+l}
```

其中：
- δ_t = r_t + γV_π(s_{t+1}) - V_π(s_t) （TD error）
- γ：折扣因子
- λ：GAE权衡因子

**λ的作用：**
- λ接近0：Ψ_t退化成TD error，高偏差低方差
- λ接近1：Ψ_t变成累积折扣奖励，低偏差高方差

### 4.3 必须实现的算法

#### 算法C1: GAE计算（从后向前DP）

**输入：**
- `rewards`: 即时奖励数组 [r_0, r_1, ..., r_T]
- `values`: Critic预测价值数组 [V_0, V_1, ..., V_T]
- `gamma`: 折扣因子（用户可调，默认0.99）
- `lam`: GAE权衡因子λ（用户可调，默认0.95）

**输出：**
- `advantages`: 优势函数数组 [A_0, A_1, ..., A_T]
- `returns`: 实际收益数组 [R_0, R_1, ..., R_T]

**计算逻辑（来源：知乎文章8.3节）：**
```javascript
function computeGAE(rewards, values, gamma, lam) {
    const length = rewards.length;
    const advantages = new Array(length).fill(0);
    let lastGaeLam = 0;
    
    // 从后向前遍历（动态规划）
    for (let t = length - 1; t >= 0; t--) {
        // 下一时刻的价值（最后一个时刻为0）
        const nextValue = (t < length - 1) ? values[t + 1] : 0;
        
        // 计算TD error: δ_t = r_t + γ·V_{t+1} - V_t
        const delta = rewards[t] + gamma * nextValue - values[t];
        
        // GAE递归公式：A_t = δ_t + γ·λ·A_{t+1}
        lastGaeLam = delta + gamma * lam * lastGaeLam;
        advantages[t] = lastGaeLam;
    }
    
    // 计算returns = advantages + values
    // 来源：知乎文章8.7节
    const returns = advantages.map((adv, i) => adv + values[i]);
    
    return { advantages, returns };
}
```

#### 算法C2: 单步计算展示

**输入：**
- `t`: 当前时刻
- `rewards`, `values`, `gamma`, `lam`: 同上
- `nextAdvantage`: 下一时刻的优势值（已计算）

**输出：**
- `calculationSteps`: 计算步骤的详细分解

**计算逻辑：**
```javascript
function getStepCalculation(t, rewards, values, gamma, lam, nextAdvantage) {
    const nextValue = (t < values.length - 1) ? values[t + 1] : 0;
    
    // TD error: δ_t = r_t + γ·V_{t+1} - V_t
    const delta = rewards[t] + gamma * nextValue - values[t];
    
    // GAE: A_t = δ_t + γ·λ·A_{t+1}
    const advantage = delta + gamma * lam * nextAdvantage;
    
    return {
        step: t,
        formulas: {
            td_error: {
                formula: "δ_t = r_t + γ·V_{t+1} - V_t",
                values: `δ_${t} = ${rewards[t].toFixed(3)} + ${gamma}×${nextValue.toFixed(3)} - ${values[t].toFixed(3)}`,
                result: delta.toFixed(3)
            },
            gae: {
                formula: "A_t = δ_t + γ·λ·A_{t+1}",
                values: `A_${t} = ${delta.toFixed(3)} + ${gamma}×${lam}×${nextAdvantage.toFixed(3)}`,
                result: advantage.toFixed(3)
            }
        },
        isPositive: advantage > 0
    };
}
```

## 五、训练模拟器 - 算法规范

### 5.1 功能描述
用户操控完整的RLHF-PPO训练过程，实时观察Loss曲线、KL散度、Reward得分等指标的变化。

### 5.2 理论来源

**来源：知乎文章第8节**

PPO的核心思想：
1. **重要性采样**：用π_old采样，用π_θ更新，提高采样效率
2. **GAE**：平衡优势函数的方差和偏差
3. **PPO-Clip**：限制策略更新幅度，防止π_θ偏离π_old过远

### 5.3 必须实现的算法

#### 算法D1: PPO-Clip Actor Loss

**来源：知乎文章第8.5节**

**核心思想：**
- 当A > 0时，ratio有上限1+ε，不能一味提升π_θ(a_t|s_t)
- 当A < 0时，ratio有下限1-ε，不能一味降低π_θ(a_t|s_t)

**输入：**
- `log_probs`: 当前Actor的log概率
- `old_log_probs`: 旧Actor的log概率（生成经验时）
- `advantages`: GAE计算的优势值
- `clip_range`: Clip范围ε（用户可调，默认0.2）

**输出：**
- `actor_loss`: Actor Loss值
- `ratio`: 重要性采样比值
- `clipped_ratio`: 裁剪后的比值

**计算逻辑：**
```javascript
function calculateActorLoss(log_probs, old_log_probs, advantages, clip_range) {
    // 计算比值 ratio = π_θ(a_t|s_t) / π_old(a_t|s_t)
    // 等价于 exp(log_probs - old_log_probs)
    const ratios = log_probs.map((lp, i) => Math.exp(lp - old_log_probs[i]));
    
    // 未裁剪的目标：ratio · A
    const unclipped = ratios.map((r, i) => r * advantages[i]);
    
    // 裁剪后的目标：clip(ratio, 1-ε, 1+ε) · A
    const clipped = ratios.map((r, i) => {
        const clipped_r = Math.max(1 - clip_range, Math.min(1 + clip_range, r));
        return clipped_r * advantages[i];
    });
    
    // PPO-Clip Loss: -min(unclipped, clipped)
    // 来源：知乎文章公式 J^{CLIP}(π_θ) = E_t[min[r_t(θ)·A, clip(r_t(θ), 1-ε, 1+ε)·A]]
    const losses = unclipped.map((u, i) => -Math.min(u, clipped[i]));
    const actor_loss = losses.reduce((a, b) => a + b, 0) / losses.length;
    
    return {
        loss: actor_loss,
        ratios: ratios,
        clipped_ratios: clipped
    };
}
```

#### 算法D2: Critic Loss计算

**来源：知乎文章第8.7节**

**核心公式：**
```
V_t^{CLIP} = clip(V_t^{new}, V_t^{old} - ε, V_t^{old} + ε)
R_t = A_t^{GAE} + V_t^{old}
L(V_φ) = E_t[max[(V_t^{new} - R_t)², (V_t^{CLIP} - R_t)²]]
```

**输入：**
- `values`: 当前Critic预测的价值 V_t^{new}
- `old_values`: 旧Critic预测的价值 V_t^{old}
- `returns`: GAE计算的returns（R_t = A_t^{GAE} + V_t^{old}）
- `clip_range_value`: Value裁剪范围（默认0.2）

**输出：**
- `critic_loss`: Critic Loss值

**计算逻辑：**
```javascript
function calculateCriticLoss(values, old_values, returns, clip_range_value) {
    // 来源：知乎文章8.7节
    // V_t^{CLIP} = clip(V_t^{new}, V_t^{old} - ε, V_t^{old} + ε)
    const values_clipped = values.map((v, i) => {
        const minVal = old_values[i] - clip_range_value;
        const maxVal = old_values[i] + clip_range_value;
        return Math.max(minVal, Math.min(maxVal, v));
    });
    
    // 计算两种Loss
    const vf_loss1 = values.map((v, i) => (v - returns[i]) ** 2);
    const vf_loss2 = values_clipped.map((v, i) => (v - returns[i]) ** 2);
    
    // 取较大值：max[(V_t^{new} - R_t)², (V_t^{CLIP} - R_t)²]
    const vf_loss = vf_loss1.map((l1, i) => Math.max(l1, vf_loss2[i]));
    
    // 求平均
    const critic_loss = vf_loss.reduce((a, b) => a + b, 0) / vf_loss.length * 0.5;
    
    return critic_loss;
}
```

#### 算法D3: Returns计算

**来源：知乎文章第8.7节**

```javascript
function calculateReturns(advantages, old_values) {
    // R_t = A_t^{GAE} + V_t^{old}
    // 来源：知乎文章8.7节公式
    return advantages.map((adv, i) => adv + old_values[i]);
}
```

#### 算法D4: 完整PPO训练步骤

**来源：知乎文章第8节**

**输入：**
- `actorParams`: Actor模型参数（简化）
- `criticParams`: Critic模型参数（简化）
- `refParams`: Reference模型参数（固定）
- `hyperparams`: {
    - `kl_ctl`: KL惩罚系数β
    - `gamma`: 折扣因子
    - `lam`: GAE权衡因子λ
    - `clip_range`: PPO clip范围ε
    - `learning_rate`: 学习率
    - `ppo_epochs`: PPO迭代轮数
}
- `batchData`: 训练批次数据（模拟）

**输出：**
- `trainingMetrics`: 训练指标

**计算逻辑：**
```javascript
function ppoTrainingStep(actorParams, criticParams, refParams, hyperparams, batchData) {
    // 1. 生成经验（使用old策略）
    const { tokens, log_probs, ref_log_probs, rewards, values } = 
        generateExperience(batchData, actorParams, criticParams, refParams, hyperparams);
    
    // 保存old值（来源：知乎文章8.2节重要性采样）
    const old_log_probs = [...log_probs];
    const old_values = [...values];
    
    // 2. 计算KL散度
    const kl_divs = calculateKL(log_probs, ref_log_probs);
    const avg_kl = kl_divs.reduce((a, b) => a + b, 0) / kl_divs.length;
    
    // 3. GAE计算优势和returns
    const { advantages, returns } = computeGAE(rewards, values, hyperparams.gamma, hyperparams.lam);
    
    // 4. PPO多轮更新（来源：知乎文章8.7节伪代码）
    let actor_loss, critic_loss;
    for (let epoch = 0; epoch < hyperparams.ppo_epochs; epoch++) {
        // 计算Actor Loss（PPO-Clip）
        const actorResult = calculateActorLoss(
            log_probs, old_log_probs, advantages, hyperparams.clip_range
        );
        actor_loss = actorResult.loss;
        
        // 计算Critic Loss
        critic_loss = calculateCriticLoss(
            values, old_values, returns, hyperparams.clip_range
        );
        
        // 更新参数（简化模拟）
        updateParams(actorParams, actor_loss, hyperparams.learning_rate);
        updateParams(criticParams, critic_loss, hyperparams.learning_rate);
        
        // 模拟参数更新后的新log_probs和values
        log_probs = simulateUpdatedLogProbs(log_probs, actorParams, hyperparams.learning_rate);
        values = simulateUpdatedValues(values, criticParams, hyperparams.learning_rate);
    }
    
    // 5. 计算平均Reward
    const avg_reward = rewards.reduce((a, b) => a + b, 0) / rewards.length;
    
    return {
        actor_loss,
        critic_loss,
        kl_div: avg_kl,
        reward: avg_reward,
        advantages: advantages
    };
}
```

```javascript
function simulateUpdatedLogProbs(old_log_probs, actorParams, learning_rate) {
    // 简化模拟：参数更新后log_probs的变化
    return old_log_probs.map(lp => {
        const noise = (Math.random() - 0.5) * learning_rate;
        return lp + noise;
    });
}

function simulateUpdatedValues(old_values, criticParams, learning_rate) {
    // 简化模拟：参数更新后values的变化
    return old_values.map(v => {
        const noise = (Math.random() - 0.5) * learning_rate;
        return v + noise;
    });
}

function updateParams(params, loss, learning_rate) {
    // 简化：Loss驱动参数变化
    params.mean -= loss * learning_rate * 0.01;
    params.std = Math.max(0.1, params.std - Math.abs(loss) * learning_rate * 0.001);
}
```

### 5.5 成就系统判定逻辑（基于真实训练结果）

```javascript
function checkAchievements(trainingHistory, currentMetrics) {
    const achievements = [];
    
    // 1. 首次收敛
    if (currentMetrics.actor_loss < 0.5 && !achievements.includes('first_converge')) {
        achievements.push('first_converge');
    }
    
    // 2. KL稳定（来源：知乎文章8.6节 KL阈值概念）
    const recentKL = trainingHistory.slice(-10).map(h => h.kl_div);
    const klStable = recentKL.every(kl => kl >= 0.1 && kl <= 0.3);
    if (klStable && !achievements.includes('kl_stable')) {
        achievements.push('kl_stable');
    }
    
    // 3. 连续优化
    const recentLosses = trainingHistory.slice(-20).map(h => h.actor_loss);
    const continuouslyImproving = recentLosses.every((loss, i) => 
        i === 0 || loss <= recentLosses[i-1]
    );
    if (continuouslyImproving && !achievements.includes('continuous_improve')) {
        achievements.push('continuous_improve');
    }
    
    // 4. 精准调参
    if (currentMetrics.actor_loss < 0.3 && currentMetrics.kl_div < 0.2 && 
        !achievements.includes('precise_tuning')) {
        achievements.push('precise_tuning');
    }
    
    // 5. 驯化大师
    if (currentMetrics.actor_loss < 0.1 && currentMetrics.kl_div < 0.2 && 
        currentMetrics.reward > 0.8 && !achievements.includes('master')) {
        achievements.push('master');
    }
    
    return achievements;
}
```
